Только актуально для формы, а не для таблицы
{Пример ниже расмотрен для таблицы в которой сразу выбераем первое поле Problem (BlockCall или DropCall), а затем вибираем тип Type данной Problem (второе поле)причем для блоков и дропов свои типы.}

Для второго поля Type указываем источник строк---Таблица или запрос:

SELECT Bl_Dr_Type.Type
FROM Bl_Dr_Type
WHERE (((Bl_Dr_Type.Problem)=[Forms]![Main]![Problem]));

Причем если так оставить то выбраное значение проблемы и его типа сохранятся  для других записей, даже если изменить в последующем Problem/
Поэтому в свойствах первого поля  Problem---События---После обновления--след процедура:

Private Sub Problem_AfterUpdate()
[Type].Requery
End Sub

REQUERY -- перезапрос, чтобы не нажимать клавишу обновить постоянно после выбора Problem
